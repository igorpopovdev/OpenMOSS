# OpenMOSS 🤖

**Multi-Agent Orchestration System** — платформа самоорганизующегося взаимодействия нескольких AI-агентов.

[English](./README.md) · [简体中文](./README_CN.md) · **Русский**

---

## 🤔 Что это?

OpenMOSS — это серверная платформа для управления задачами и оркестрации AI-агентов. Она позволяет нескольким AI-агентам (Планировщик, Исполнитель, Рецензент, Патрульный) совместно работать над задачами в цикле **планирование → выполнение → рецензирование → патрулирование**.

Агенты могут быть реализованы на любом языке — главное, чтобы они поддерживали HTTP API и могли работать с `task-cli.py`.

## 👥 Роли агентов

| Роль | Описание |
|------|---------|
| **Планировщик (Planner)** | Разбивает задачу на подзадачи, планирует порядок |
| **Исполнитель (Executor)** | Выполняет конкретные подзадачи |
| **Рецензент (Reviewer)** | Проверяет результаты, принимает или отправляет на доработку |
| **Патрульный (Patrol)** | Мониторит процессы, выявляет аномалии |

## 🚀 Быстрый старт

### 1. Запуск сервера

```bash
pip install -r requirements.txt
python server.py
```

Откройте **http://localhost:6565** в браузере и следуйте указаниям мастера первоначальной настройки.

### 2. Подключение агента

Каждый агент использует `task-cli.py` для взаимодействия с сервером:

```bash
python task-cli.py \
  --api-key YOUR_API_KEY \
  --agent-id YOUR_AGENT_ID \
  --server http://localhost:6565
```

### 3. Первые шаги

1. Создайте задачу через веб-интерфейс
2. Агент-планировщик автоматически получит уведомление и разобьёт её на подзадачи
3. Исполнители получат подзадачи и начнут работу
4. Рецензент проверит результаты
5. Патрульный будет следить за аномалиями

## 🔌 API

### Endpoints

| Метод | Путь | Описание |
|-------|------|---------|
| `POST` | `/tasks` | Создать задачу |
| `GET` | `/tasks/{id}` | Получить задачу |
| `PUT` | `/tasks/{id}/status` | Обновить статус задачи |
| `POST` | `/sub-tasks` | Создать подзадачу |
| `GET` | `/agents` | Список агентов |
| `POST` | `/agents/register` | Регистрация агента |
| `GET` | `/scores/leaderboard` | Табло рейтинга |

Полная документация API — в [API.md](./docs/API.md).
