# grok-search — Документация по архитектуре (RU)

## 1. Цели

- Внешнее имя — **grok-search**; контекстный поиск — только как **внутренняя модульная возможность** (не предоставляется отдельно)
- Использование **grok-4.20-beta** для генерации ответов с "самостоятельным поиском модели"
- При сбое/rate limit/неудовлетворительном качестве — автофолбэк на **grok-4.1-thinking**
- Опциональный контекстный поиск (внутренний модуль `tools/grok_search/context.py`) для продолжения диалога
- **Промпт содержит только "ищи и возвращай источники"** — без упоминания внутреннего поиска

## 2. Поток вызовов (4.2 в приоритете, фолбэк на 4.1)

1. **Модуль контекстного поиска** `route()` генерирует `task_id` и флаг необходимости контекста
2. **Модуль контекстного поиска** `build()` генерирует историю сообщений
3. Вызов **grok-4.20-beta**
4. Проверка триггера фолбэка (см. ниже)
5. Вызов **grok-4.1-thinking**
6. **Модуль контекстного поиска** `record()` записывает user/assistant + meta

## 3. Единый системный промпт

```
Вы должны активно искать в сети и возвращать доказательства.
Требования:
1) Минимум 2 источника (URL, время публикации, цитата)
2) Не придумывайте источники; если не нашли — напишите "Не найдено"
3) Сначала доказательства, потом вывод
Вывод JSON:
{
  "sources":[{"title":"","url":"","published_at":"","quote":""}],
  "claims":[{"statement":"","source_index":[0]}],
  "answer":"…",
  "unknowns":["…"],
  "confidence":0-1
}
```

> Рекомендуется добавить `stream:false` в параметрах, чтобы получить JSON целиком.

## 4. Триггеры фолбэка / сбоя

- HTTP не 200 / таймаут / 429
- `sources` пуст или меньше 2
- Любой source без `url / published_at / quote`
- `answer` пуст
- Явно заявлено "не могу искать/источники не найдены" без доступных доказательств

## 5. Структура вывода (единый JSON)

```json
{
  "sources":[{"title":"","url":"","published_at":"","quote":""}],
  "claims":[{"statement":"","source_index":[0]}],
  "answer":"…",
  "unknowns":["…"],
  "confidence":0-1
}
```

## 6. Стратегии контекста (внутренний модуль)

- `stateless`: без контекста
- `auto`: с контекстом только по ключевым словам (рекомендуемое значение по умолчанию)
- `always`: всегда с контекстом (для сценариев продолжения)

## 7. Конфигурация и ключи

### 7.1 Переменные окружения

- **Предпочтительно**:
  - `GROK_BASE_URL`
  - `GROK_API_KEY`
- **Обратная совместимость (текущая конфигурация)**:
  - `GAPI_CUSTOM_BASE_URL`
  - `GAPI_CUSTOM_API_KEY`
- **Примечание**: текущая среда использует `GAPI_CUSTOM_BASE_URL / GAPI_CUSTOM_API_KEY` из `.env` как base/key Grok (эквивалент `GROK_*`).

### 7.2 macOS Keychain

- Service: `openclaw-grok`
- Accounts: `base-url`, `api-key`, `model`

## 8. Эндпоинты и формат запросов

- OpenAI-совместимый интерфейс: `POST /v1/chat/completions`
- Ключевые поля: `model`, `messages`, `stream:false`, `temperature`

## 9. Запись метаинформации (внутренний модуль)

- `model`, `fallback_reason`, `latency_ms`
- Сохраняются только **выводы + цитаты**, без длинных текстов
