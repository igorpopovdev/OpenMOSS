# Local Web Search — SKILL_RU

Используйте локальный Responses API через `web_search` для получения веб-результатов и цитат.

## Быстрый старт

1. Убедитесь, что переменные окружения заданы ИЛИ настроены Keychain (см. `references/CONFIG.md`).
2. Запустите скрипт:

```bash
/usr/bin/python3 {baseDir}/scripts/local_web_search.py --query "<ваш запрос>" --mode live
```

Опционально (явный путь .env):

```bash
/usr/bin/python3 {baseDir}/scripts/local_web_search.py \
  --env-file ~/.openclaw/workspace/.env \
  --query "<ваш запрос>" \
  --mode cached
```

Таймаут по умолчанию — 1200с. Переопределите через `--timeout` или `LOCAL_160_TIMEOUT`.
Примечание: ответы web_search могут быть медленными; 5–10 минут — обычное дело, до ~20 минут возможно. Не считайте это ошибкой, если не превышен таймаут.

## Обработка вывода

- По умолчанию: одно предложение-сводка + до 5 цитат.
- Используйте `--json` для получения полного вывода инструмента.
- Используйте `--mode live|cached|disabled` для управления внешним доступом.

## Когда использовать

- Для веб-поиска через local-160.
- Для запланированного мониторинга новостей через local-160.

## Устранение неполадок

- 401/403: ключ отсутствует или недействителен.
- 5xx: локальный шлюз недоступен или неправильный base URL.
